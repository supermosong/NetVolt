Computer Networks

Learn how billions of devices talk to each other, how a message travels from your laptop to a server on the other side of the world in milliseconds, and what TCP/IP actually means.

Beginner ~20 min read 5-Question Quiz

What is a Computer Network?

A network is two or more devices connected together so they can share data and resources.

When your laptop sends a file to your phone over Wi-Fi, or when you stream a video from a server thousands of kilometres away, you are using a network. The internet itself is simply the world's largest network — a network of networks.

Networks let devices share things: files, printers, internet connections, and applications. Without networking, every computer would be an island — useful on its own, but unable to communicate with the outside world.

Key idea

Every device on a network has a unique address — its IP address — just like a postal address for a house. This is how data finds its way to exactly the right device out of billions connected globally.

LAN

Local Area Network. Covers a small area — a home, office, or school. All devices share one internet connection and can communicate directly with each other.

WAN

Wide Area Network. Spans cities, countries, or the entire globe. The internet is the world's largest WAN — it connects millions of smaller LANs together.

MAN

Metropolitan Area Network. Sits between a LAN and a WAN — covers a city or campus. Often used to link multiple offices of the same organisation.

LAN vs WAN — Up Close

Understanding the difference between local and wide-area networks is the foundation of all networking knowledge.

Imagine your home as a small village. Every house (device) in the village can talk to every other house easily — that's your LAN. The road connecting your village to the rest of the country — that's the WAN (the internet). Your router is the village gate: it controls traffic in and out.

Packets & Routing

Data doesn't travel across the internet in one huge chunk — it is broken into tiny labelled envelopes called packets.

Imagine sending a 1,000-page book through the post. Instead of shipping one giant parcel, you tear out every page, put each in a separate envelope labelled with the destination address and a page number, and send them all. They may take different routes and arrive in any order — but the recipient reassembles them in sequence. That is exactly how internet data works.

What's inside a packet?

Every packet carries a header (the envelope label) containing: the sender's IP address, the destination IP address, a sequence number so the receiver can reorder them, and error-checking data. The payload is the actual content — a fragment of your email, photo, or video.

How Packets Travel

1. Fragmentation

Your device breaks the data into small packets (typically up to 1,500 bytes each) and numbers them so they can be reassembled in the right order at the other end.

2. Routing

Each packet is forwarded by a series of routers across the internet. Every router reads the destination IP address and picks the best next hop — like a relay race across the globe.

3. Reassembly

The destination device collects all packets, checks for errors, requests any missing ones to be resent, and reassembles them into the original data — all in milliseconds.

TCP/IP & Protocols

A protocol is a set of agreed rules. TCP/IP is the foundational rulebook that every device on the internet follows.

For two computers to communicate they must agree on: how to address each other, how to break data into packets, how to reassemble them, and what to do if something goes wrong. These agreements are called protocols. TCP/IP is not one protocol but a family of them, organised into four layers — each layer handling a different part of the job.

The TCP/IP Layer Stack

Data passes down the stack when sending and up the stack when receiving. Click each layer to learn what it does.

The layer your programs use directly. When your browser asks for a webpage it uses HTTP or HTTPS. When it looks up a domain name it uses DNS. When your email client sends mail it uses SMTP. This layer defines what is being requested, not how to deliver it — that's the lower layers' job.

Handles end-to-end delivery between two programs. There are two main protocols here:

  • TCP (Transmission Control Protocol) — guarantees delivery. If a packet is lost it is resent. Used for web browsing, email, file downloads — anything where accuracy matters.
  • UDP (User Datagram Protocol) — faster but no delivery guarantee. Used for video calls, live streams, and games where a tiny bit of lost data is acceptable but speed is critical.

The IP (Internet Protocol) layer handles addressing and routing. It stamps every packet with the sender and destination IP addresses, then routers across the internet use those addresses to forward the packet to its destination. Think of IP as the postal address written on an envelope — without it, no one knows where to send the data.

The physical layer — the actual cables, radio waves, or fibre optic light pulses that carry the raw bits. This layer converts digital 1s and 0s into electrical signals (Ethernet cable), radio waves (Wi-Fi), or pulses of light (fibre optic). It also handles the short-range addressing between devices on the same local network using MAC addresses.

Key Internet Services

Three services you use every time you open a browser — even if you've never heard their names.

DNS

Domain Name System — the internet's phonebook. Computers communicate using IP addresses (e.g. 142.250.80.46), but humans prefer names like google.com. DNS translates one into the other automatically, every time you type a web address.

Real-world analogy

DNS is like asking a receptionist "What's the phone number for Google?" instead of memorising the number yourself.

Not Encrypted

HTTP

HyperText Transfer Protocol — the language browsers use to request and receive web pages. When you type a URL your browser sends an HTTP request; the server sends an HTTP response containing the page. Plain HTTP sends data in the open — anyone on the network can read it.

Encrypted

HTTPS

HTTP Secure — HTTP with encryption added via TLS (Transport Layer Security). All data between your browser and the server is scrambled so only those two parties can read it. Always look for the padlock icon in your browser's address bar.

What Happens When You Type a Website Address

This happens in under 20 milliseconds

The full DNS lookup chain above typically completes in 10–50 ms — and results are cached (saved) by your browser and ISP, so frequently-visited sites skip most of these steps and resolve even faster.
